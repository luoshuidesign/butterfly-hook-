## 〇、为什么这个计划是对的

```
论证起点:

问题: 世界上有无数好设计。但普通人只能看见表面。
      设计师知道为什么好。产品经理知道为什么卖。
      但普通人不知道。学生不知道。创业者不知道。

Hook解决的就是这个:
  让不可见 → 可见
  让不可懂 → 可懂
  让不可学 → 可学

市场验证:
  Clone Architect 9.3k★ → 证明"设计提取"有需求
  Taste Lab PH#1           → 证明"设计推理"有需求
  Figma $20B估值           → 证明"设计工具"有市场
  设计教育市场 $40B        → 证明"设计学习"有市场

Hook不是跟它们竞争。Hook是它们的"认知层"。
  Figma = 设计的"手"（制作工具）
  Hook  = 设计的"眼"（看见本质）

手和眼不是竞争关系。是互补关系。
```

## 一、Phase 1: 内测积累 (Day 1-30)

### 1.1 目标

```
核心指标:
  知识库: 90个站点
  文章: 10篇
  设计模式: 自动聚类≥5种
  六维分析: 每站一份

为什么是90?
  30天 × 3站/天 = 90
  90站足够产生统计学意义的设计趋势
  < 90 → 样本不够·趋势不可靠
  > 90 → 边际收益递减
```

### 1.2 每日执行

```
每日任务 (约30分钟):

  08:00  打开Hook·钩取今天的3个目标站
  08:05  Hook自动生成六维分析报告
  08:15  快速浏览报告·标注有趣发现
  08:20  存入知识库·自动聚类
  20:00  每周五写一篇综合文章

钩取计划 (30天·5类·每类6天):

  第1-6天:   科技公司
    Apple / Stripe / Linear / Vercel / Figma / Notion
    目标: 理解科技美学的共性

  第7-12天:  电商平台
    Shopify / Amazon / 拼多多 / 京东 / Airbnb / Uber
    目标: 理解转化的设计逻辑

  第13-18天: 创作者平台
    Dribbble / Behance / 站酷 / Substack / Patreon / Gumroad
    目标: 理解创作者的商业模式

  第19-24天: 金融科技
    Robinhood / 支付宝 / Revolut / Stripe / Coinbase / Wise
    目标: 理解信任如何通过设计建立

  第25-30天: 自由选择
    你最感兴趣的·最想学习的·最好奇的
    目标: 满足自己的好奇心·发现意外规律
```

### 1.3 10篇文章计划

```
Week 1: "我用Hook解剖了Apple的设计——不是好看·是哲学"
        → 论证: Apple的设计哲学 = 极简=自信=高端
        → 数据: Hook提取的配色·字体·间距·导航DNA

Week 1: "Stripe为什么感觉那么'开发者'？Hook发现了答案"
        → 论证: Stripe的文化 = 代码美学 + 极客信仰

Week 2: "30个电商网站的转化秘密——Hook的数据告诉你"
        → 论证: CTA颜色·定价展示·信任信号的统计规律

Week 2: "日本设计 vs 美国设计——Hook的文化对比"
        → 论证: 克制 vs 张扬·留白 vs 满屏·数据支撑

Week 3: "2026设计趋势——不是观点·是90个站的数据"
        → 论证: 深色模式占比·玻璃态频率·AI界面趋势

Week 3: "为什么有的设计让你信任？Hook的认知分析"
        → 论证: 信任信号·认知负荷·注意力引导的规律

Week 4: "从Hook看定价心理学——$9.9的秘密"
        → 论证: 价格锚定·套餐设计·折扣展示的规律

Week 4: "Hook是什么·为什么我做了一个'看见'引擎"
        → 论证: Hook的哲学·六维·使命

Week 5: "一个设计师·一个开发者·一个产品经理——Hook帮了他们什么"
        → 论证: 三个用户故事·真实场景

Week 6: "Hook的知识库公开——90个站·6个维度·全部数据"
        → 论证: 开放数据·邀请社区·共建生态
```

### 1.4 成功标准

```
Phase 1 成功 = 三个条件同时满足:

  ✅ 知识库≥90站·自动聚类≥5种
  ✅ 10篇文章≥3篇被转载/引用

## 三、Phase 3: 验证付费 (Day 60-90)

### 3.1 核心假设

```
假设: 有人愿意为"设计洞察"付费

验证方式: 最轻量的付费产品

不是SaaS (开发成本高·周期长)
不是课程 (内容生产成本高)
是: 设计洞察月报 (PDF·$9/期)

为什么是月报:
  ✅ 零开发 → Hook已经能生成所有数据
  ✅ 零内容生产 → Hook的数据+AI总结 = 报告
  ✅ 定价低 → $9降低决策门槛
  ✅ 验证快 → 一个月就能知道结论
```

### 3.2 产品设计

```
产品: Hook 设计洞察月报

内容 (每期约20页):
  ① 本月20个精选网站的六维分析
  ② 设计趋势可视化 (配色·字体·布局·动效)
  ③ 行业对比 (科技vs电商vs金融)
  ④ 深度案例 (1个网站的完整设计哲学解剖)
  ⑤ 实用建议 (可操作的设计改进方案)

定价:
  单期: $9
  年费: $59 (省$49)
  终身: $149 (限前100名)

渠道:
  Gumroad (全球)
  爱发电 (国内)
  小报童 (国内·内容付费)

成本:
  生产: ~2小时/期 (Hook自动生成·手动精选编辑)
  渠道: Gumroad抽成10%
  净利: $8.1/期
```

### 3.3 验证指标

```
关键指标:
  首月销量: ≥20份
  续订率: ≥50% (第二个月还有人买)
  用户反馈: ≥5条详细评论

Phase 3 成功 = 两个条件都满足:

  ✅ 首月销量≥20份 ($162收入)
  ✅ 续订率≥50%

不满足 → 重新思考付费模式·不进入Phase 4
满足 → 验证了"有人愿意为设计洞察付费"·进入Phase 4
```

## 四、Phase 4: SaaS上线 (Day 90-270)

### 4.1 产品

```
hook.design

核心功能:
  ① 在线钩取 → URL→10秒出DNA·无需安装
  ② 设计图库 → 公共+私人·标签·搜索·收藏
  ③ 六维报告 → 自动生成·一键导出PDF
  ④ 定时监控 → 竞品变化·自动告警·diff报告
  ⑤ 团队协作 → 共享图库·评论·权限

技术栈:
  前端: Next.js 16 + Tailwind v4 + shadcn/ui
  后端: Node.js (复用Hook 11脚本)
  数据库: PostgreSQL + Redis (缓存)
  任务队列: Bull (定时钩取)
  部署: Vercel (前端) + Railway (后端)
  存储: Cloudflare R2 (截图+报告)

MVP开发:
  2个月·1个全职开发者
  核心: 在线钩取 + 图库 + 基础报告
  不做的: 定时监控·团队协作 (Phase 4.2)

## 六、风险矩阵

```
风险1: 没人用
  概率: 低
  影响: 高
  对策: Phase 1用内容验证兴趣·Phase 2用开源验证需求
       如果Phase 2的Stars <100 → 重新评估定位

风险2: 被大公司复制
  概率: 中
  影响: 中
  对策: 数据壁垒 (知识库) + 社区壁垒 + 开源生态
       他们可以复制功能·复制不了数据和社区

风险3: 法律问题
  概率: 低
  影响: 高
  对策: 只提取公开CSS·不破解·不越权
       明确的fair use声明·不提取版权内容

风险4: 付费意愿不足
  概率: 中
  影响: 中
  对策: Phase 3用$9低价验证·如果不行转向广告/赞助模式

风险5: 技术维护成本
  概率: 中
  影响: 低
  对策: Hook核心300行·维护成本极低·反爬更新是最大开销
```
