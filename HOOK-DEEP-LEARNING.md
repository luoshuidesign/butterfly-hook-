Hook = 永远在训练的神经网络

## 三、注意力计算 → Hook的焦点提取

```
Transformer的注意力:       Hook的暗行通道:

Q (Query)  = 我想找什么    = 用户意图·设计维度
K (Key)    = 我有什么      = 网页元素·CSS属性  
V (Value)  = 实际内容      = 计算后的精确值

Attention(Q,K,V) = softmax(QK^T/√d) × V

Hook的"注意力":
  不是均匀扫描所有元素
  是给关键设计元素分配更高的"注意力权重"
  
  H1的fontSize    → 权重 0.95 (极高)
  普通p标签的color → 权重 0.30 (普通)
  div的padding    → 权重 0.15 (低)
  
  30条暗行通道 = 30个注意力头
  每条通道关注不同的设计维度
  Multi-Head Attention = Multi-Dimension Hook

Hook的每一条暗行通道都像一个注意力头:
  P1  → 关注渲染后的CSS
  P2  → 关注伪元素
  P6  → 关注动画
  P14 → 关注布局几何
  ...
  
  30个头·并行关注·最后融合
```

## 六、为什么这很重要

```
深度学习教会我们:

① 深度 > 宽度
   不是更多通道·是更深的层次
   Hook: 不是100条浅通道·是30条深通道×6层处理

② 注意力 > 均匀
   不是所有元素同等重要
   Hook: 给关键设计元素高权重·忽略噪音

③ 反馈 > 单向
   不是一次提取就结束
   Hook: 每次验证→反馈→进化→下次更准

④ 预训练+微调 > 从零训练
   不是每个网站从零分析
   Hook: 知识库预训练·新站点微调

⑤ 多头 > 单头
   不是一个角度看设计
   Hook: 30个头·30个角度·融合理解
```
